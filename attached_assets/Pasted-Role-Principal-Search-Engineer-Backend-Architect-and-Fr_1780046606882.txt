====================================================
PART 1 — SEARCH FOUNDATION FIXES
================================

Current Problems:

* Exact-match dependent behavior
* Weak incremental search
* Poor discovery experience
* Empty results while typing
* Search feels unreliable

REQUIRED IMPLEMENTATION:

1. Real-Time Incremental Search
   Implement proper search-as-you-type behavior.

Requirements:

* partial matching
* prefix matching
* incremental results while typing
* fuzzy matching tolerance
* debounce optimization
* responsive loading states

Behavior Examples:
Typing:

* "algo"
  should return:
* "Algorithms"
* "Advanced Algorithms"
* "Algorithm Analysis"

Typing:

* "datab"
  should return:
* "Database Systems"
* "Databases 101"

Minor typos should still produce useful results.

2. Search Scope
   Search across:

* document title
* filename
* tags
* categories
* descriptions
* metadata fields
* uploader names if indexed already

3. Search Ranking Logic
   Search results should prioritize:

* exact matches
* partial matches
* metadata relevance
* engagement
* popularity
* recency

4. Search UX Improvements
   Improve frontend UX:

* instant feedback
* loading/searching indicators
* no-results state
* highlighted matching text
* smoother mobile search behavior

5. Performance Requirements

* avoid excessive API calls
* debounce requests properly
* optimize query performance
* maintain responsiveness with larger datasets
* preserve current search architecture if possible
