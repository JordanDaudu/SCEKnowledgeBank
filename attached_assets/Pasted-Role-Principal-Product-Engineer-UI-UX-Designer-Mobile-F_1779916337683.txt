Part A — Add restrained semantic visual system

1. Add or improve CSS/theme tokens in `index.css`
   Keep the existing green/cream academic style, but add restrained semantic helpers for:

* status open
* status in progress
* status fulfilled
* status closed
* status draft
* status pending review
* status approved/published
* status rejected
* course tag
* material type tag
* soft card hover
* subtle section accents

Preferred semantic color direction:

* primary/brand: forest green
* background: warm cream
* neutral text: deep green-gray
* open/info: muted blue or blue-gray
* in-progress/pending: amber
* fulfilled/approved: emerald
* closed/archived: slate
* rejected/destructive: muted red
* draft: soft blue-gray
* course tags: sage/green-tinted
* material tags: warm neutral

Do not over-color the UI. Use color mainly in badges, icon tiles, accent borders, alerts, and hover states.

2. Prefer reusable helpers
   If practical, create small shared helpers/components instead of repeating class names everywhere:

* `StatusBadge` should support document statuses with clear semantic colors.
* Add request status styling helper inside `requests.tsx` or a small reusable component if clean.
* Keep implementation simple.
